# Business Processes

## Overview

Документ описывает ключевые бизнес-процессы М.И.Т.А., включая процессы продаж, производства услуг, поддержки и управления.

---

## Process: Лидогенерация (Основной процесс)

### Owner
Отдел продаж (ГЦК) + IT-отдел + КЦ

### Trigger
Поступление новых лидов из внешних источников (парсинг, сбор данных)

### Steps

**Этап 1: Сбор и загрузка лидов**
```
[IT-аналитик]
1. Получение данных из внешних источников
2. Обработка и валидация данных
3. Загрузка лидов в Битрикс24
4. Проверка доступности для отдела продаж
```

**Этап 2: Первичный обзвон (ГЦК)**
```
[Отдел продаж ГЦК]
1. Получение лида из Битрикс24
2. Холодный звонок
3. Презентация услуги лидогенерации
4. Предложение Платного теста (10 000 руб.)
5. При согласии → выставление счёта
```

**Этап 3: Платный тест**
```
[Клиент] → Оплата 10 000 руб.
         ↓
[Отдел тестов]
1. Получение заказа
2. Обработка 100 звонков
3. Фиксация результатов (конверсия ~10%)
4. Передача результата клиенту
5. Уведомление менеджера о завершении
```

**Этап 4: Продажа основного тарифа**
```
[Отдел продаж ГЦК]
1. Контакт с клиентом после теста
2. Презентация основного тарифа (148 000 руб. / 1000 звонков)
3. При согласии → заключение договора, оплата
```

**Этап 5: Основная обработка**
```
[КЦ (Колл-центр)]
1. Получение заказа на 1000 звонков
2. Обзвон по базе
3. Квалификация лидов
4. Передача готовых лидов клиенту
5. Отчётность
```

### Output
- Готовые "тёплые" лиды для клиента
- Отчёт по результатам обзвона
- Оплата от клиента

### Metrics

| Метрика | Значение |
|---------|----------|
| **Конверсия из звонка в лид** | ~10% |
| **Конверсия из теста в тариф** | ~30-50% (оценка) |
| **Время обработки (тест)** | 1-3 дня |
| **Время обработки (тариф)** | 5-10 дней |
| **Средний чек** | 148 000 руб. |

### Tools
- Битрикс24 (CRM)
- Телефония (запись разговоров)
- Скрипты продаж
- Скрипты автоматизации (IT-отдел)

## Process: Рекрутинг

### Owner
Отдел рекрутинга + КЦ

### Trigger
Заявка от клиента на подбор персонала

### Steps

**Этап 1: Приём заявки**
```
[Клиент] → Заявка на подбор
         ↓
[Отдел продаж]
1. Выяснение требований к кандидатам
2. Определение объёма (10/100/300/1000)
3. Выбор тарифа
4. Выставление счёта
```

**Этап 2: Подготовка**
```
[Отдел рекрутинга]
1. Составление сценария обзвона
2. Обучение операторов
3. Подготовка скриптов
```

**Этап 3: Обзвон и отбор**
```
[КЦ]
1. Массовый обзвон потенциальных кандидатов
2. Первичное интервью
3. Проверка соответствия требованиям
4. Фиксация результатов
```

**Этап 4: Передача результата**
```
[Отдел рекрутинга]
1. Формирование базы отобранных кандидатов
2. Передача клиенту
3. Отчётность
```

### Output
- База отобранных кандидатов (10/100/300/1000)
- Отчёт по результатам подбора

### Metrics

| Метрика | Значение |
|---------|----------|
| **Срок подбора** | Зависит от тарифа |
| **Конверсия в кандидата** | ~10-20% (оценка) |
| **Стоимость за кандидата** | 1 000 — 1 300 руб. |

### Tools
- Битрикс24
- Телефония
- Скрипты интервью

## Process: Авито

### Owner
Отдел продвижения Авито

### Trigger
Заявка от клиента на продвижение в Авито

### Steps

**Этап 1: Приём заявки**
```
[Клиент] → Заявка на продвижение
         ↓
[Отдел продаж]
1. Определение региона (МСК/СПБ или другие)
2. Определение количества регионов (<4 или ≥5)
3. Выбор тарифа (30 000 или 40 000 руб./мес)
4. Выставление счёта
```

**Этап 2: Запуск**
```
[Отдел Авито]
1. Получение информации от клиента (товары/услуги)
2. Создание объявлений
3. Публикация объявлений
4. Оптимизация под поиск
```

**Этап 3: Ведение**
```
[Отдел Авито]
1. Регулярное обновление объявлений
2. Ответы на сообщения
3. Модерация отзывов
4. Отчётность клиенту (ежемесячно)
```

### Output
- Активные объявления на Авито
- Заявки от потенциальных клиентов
- Ежемесячный отчёт

### Metrics

| Метрика | Значение |
|---------|----------|
| **Время запуска** | 1-2 дня |
| **Количество объявлений** | Зависит от тарифа |
| **Стоимость лида** | Зависит от ниши |

### Tools
- Авито (платформа)
- Инструменты аналитики Авито

## Process: Call-центр (Аутсорс)

### Owner
КЦ (Колл-центр)

### Trigger
Заявка от клиента на обзвон базы

### Steps

**Этап 1: Приём заявки**
```
[Клиент] → Предоставление базы номеров
         ↓
[Отдел продаж]
1. Определение объёма звонков
2. Выбор скрипта (или разработка нового)
3. Выставление счёта
```

**Этап 2: Подготовка**
```
[КЦ]
1. Загрузка базы в телефонию
2. Обучение операторов
3. Тестовый обзвон (10-20 номеров)
```

**Этап 3: Обзвон**
```
[Операторы КЦ]
1. Обзвон по базе
2. Работа по скрипту
3. Фиксация результатов
4. Квалификация ответов
```

**Этап 4: Отчётность**
```
[КЦ]
1. Формирование отчёта
2. Передача результатов клиенту
3. Рекомендации по улучшению
```

### Output
- Отчёт по результатам обзвона
- Список заинтересованных клиентов
- Запись разговоров (по запросу)

### Metrics

| Метрика | Значение |
|---------|----------|
| **Звонков в день (оператор)** | ~80-100 |
| **Конверсия в интерес** | ~5-15% (зависит от базы) |
| **Время запуска** | 1-3 дня |

### Tools
- Телефония
- Битрикс24
- Скрипты продаж

## Process: IT-поддержка и автоматизация

### Owner
IT-отдел

### Trigger
Поступление задачи (плановой или спонтанной)

### Steps

**Этап 1: Загрузка лидов (ежедневно)**
```
[IT-аналитик]
1. Проверка источников лидов
2. Запуск скриптов парсинга/сбора
3. Обработка данных
4. Загрузка в Битрикс24
5. Проверка доступности для отдела продаж
```

**Этап 2: Мониторинг автоматизаций**
```
[IT-аналитик]
1. Проверка работоспособности скриптов
2. Мониторинг ошибок
3. Оперативное исправление проблем
```

**Этап 3: Спонтанные задачи**
```
[Сотрудник] → Заявка на помощь
         ↓
[IT-аналитик]
1. Получение запроса
2. Приоритизация
3. Решение задачи (помощь с ПК, Битрикс24, ПО)
4. Закрытие задачи
```

**Этап 4: Резервное копирование**
```
[IT-аналитик]
1. Еженедельное/ежемесячное копирование
2. Проверка целостности данных
3. Хранение резервных копий
```

**Этап 5: Отчётность**
```
[IT-аналитик]
1. Сбор данных о проделанной работе
2. Формирование отчёта
3. Передача руководству (по запросу)
```

### Output
- Стабильная загрузка лидов
- Рабочие автоматизации
- Решённые технические задачи
- Резервные копии данных

### Metrics

| Метрика | Значение |
|---------|----------|
| **Бесперебойность загрузки** | 99%+ времени |
| **Время реакции на задачу** | По мере поступления |
| **Резервное копирование** | Еженедельно/ежемесячно |

### Tools
- Python (скрипты)
- SQLite (базы данных)
- Docker (контейнеризация)
- Битрикс24 (API)
- Qwen Code (AI-ассистент)

## Process: Обучение сотрудников

### Owner
IT-отдел + HR-отдел

### Trigger
Необходимость обучения новым инструментам/процессам

### Steps

**Этап 1: Выявление потребности**
```
[Сотрудник / Руководитель]
1. Определение навыков для обучения
2. Заявка в IT/HR
```

**Этап 2: Подготовка материалов**
```
[IT-отдел]
1. Создание инструкций
2. Подготовка презентаций
3. Демонстрационные материалы
```

**Этап 3: Проведение обучения**
```
[IT-аналитик]
1. Презентация инструмента
2. Практическая демонстрация
3. Ответы на вопросы
```

**Пример:** Обучение использованию ИИ (Gemini, Leonardo AI, Notebook LLM)

### Output
- Обученные сотрудники
- Инструкции и материалы

### Metrics

| Метрика | Значение |
|---------|----------|
| **Частота обучения** | По необходимости |
| **Охват** | Заинтересованные сотрудники |

## Process: Встречи и коммуникация

### Owner
Коммерческий директор + Руководители отделов

### Trigger
Необходимость обсуждения вопросов

### Steps

**Плановые встречи:**
- Встречи с коммерческим директором (по необходимости)
- Обсуждение инициатив и улучшений

**Спонтанные встречи:**
- По инициативе руководителя
- По инициативе сотрудника

### Output
- Принятые решения
- План действий

### Metrics

| Метрика | Значение |
|---------|----------|
| **Частота** | По необходимости (редко) |
| **Формат** | Лично / Онлайн |

## Process: Найм сотрудников

### Owner
HR-отдел + Руководители отделов

### Trigger
Открытие вакансии

### Steps

**Этап 1: Открытие вакансии**
```
[Руководитель отдела]
1. Определение требований
2. Заявка в HR-отдел
```

**Этап 2: Поиск кандидатов**
```
[HR-отдел]
1. Размещение вакансии
2. Первичный отбор резюме
3. Назначение собеседований
```

**Этап 3: Собеседование**
```
[HR + Руководитель]
1. Интервью с кандидатом
2. Оценка навыков
3. Принятие решения
```

**Этап 4: Оформление**
```
[HR-отдел]
1. Оформление документов
2. Выход на работу
3. Адаптация
```

### Пример текущей вакансии:
- **Специалист по AI** (в процессе найма)

### Output
- Новый сотрудник в штате

### Metrics

| Метрика | Значение |
|---------|----------|
| **Срок закрытия вакансии** | 2-4 недели (оценка) |
| **Каналы поиска** | Сайты вакансий, рекомендации |

## Process: Контроль качества

### Owner
Руководители отделов

### Trigger
Регулярный контроль / Жалобы клиентов

### Steps

**Для КЦ:**
```
[Руководитель КЦ]
1. Прослушивание записей разговоров
2. Оценка по чек-листу
3. Обратная связь оператору
4. Корректирующие действия
```

**Для отдела продаж:**
```
[Коммерческий директор / РОП]
1. Анализ конверсии
2. Прослушивание звонков
3. Коучинг менеджеров
```

**Для IT-отдела:**
```
[Коммерческий директор]
1. Проверка стабильности загрузки лидов
2. Анализ выполненных задач
3. Обратная связь
```

### Output
- Отчёт о качестве
- План улучшений

### Metrics

| Метрика | Значение |
|---------|----------|
| **Частота контроля** | Еженедельно/ежемесячно |
| **Формат** | Внутренний аудит |

## SLA (Service Level Agreement)

### Внутренние SLA

| Процесс | Срок | Ответственный |
|---------|------|---------------|
| **Загрузка лидов** | Ежедневно, бесперебойно | IT-отдел |
| **Обработка заявки клиента** | 1-3 дня | Отдел продаж |
| **Запуск Платного теста** | 1 день после оплаты | Отдел тестов |
| **Обработка теста (100 звонков)** | 1-3 дня | Отдел тестов |
| **Обработка тарифа (1000 звонков)** | 5-10 дней | КЦ |
| **Запуск Авито** | 1-2 дня | Отдел Авито |
| **IT-поддержка** | По мере поступления | IT-отдел |
| **Резервное копирование** | Еженедельно/ежемесячно | IT-отдел |
