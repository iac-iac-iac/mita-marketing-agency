## 📊 План рефакторинга

### Phase 1: Критические исправления (1-2 недели, 8-12 часов)

1. **Валидация Bitrix24 webhook URL** — проверка https:// и домена
2. **Timeout для fetch запросов** — AbortController с 5s timeout
3. **Исправить ID счётчика аналитики** — использовать .env.local
4. **Добавить security headers** — CSP, HSTS, X-Frame-Options
5. **Интегрировать useUTM в ContactForm** — автозаполнение меток
6. **Очистить проект** — удалить пустые директории

### Phase 2: Улучшение архитектуры (2-3 недели, 20-30 часов)

1. **Кэширование для MDX** — Next.js cache() или in-memory с TTL
2. **Рефакторинг CMS функций** — устранить дублирование blog.ts/cases.ts
3. **Rate limiting на Redis** — замена in-memory на Vercel KV/Upstash
4. **Разделить главную страницу** — вынести секции в компоненты
5. **Базовый Navigation компонент** — объединить Header/MainHeader
6. **Настроить Jest** — конфигурация для запуска тестов
7. **Тесты для API** — валидация, rate limiting, CSRF

### Phase 3: Оптимизация (1-2 недели, 12-18 часов)

1. **Bundle analyzer** — @next/bundle-analyzer
2. **Lazy loading для видео** — Intersection Observer
3. **Font optimization** — display: optional
4. **Storybook** — для UI компонентов
5. **Pre-commit хуки** — Husky с lint-staged
6. **Sentry** — error tracking
