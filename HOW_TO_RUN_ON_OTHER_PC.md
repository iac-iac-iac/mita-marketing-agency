# 🚀 Инструкция по запуску сайта Direct-line на другом компьютере

## Требования

- **Node.js:** версии 18 или выше
- **npm:** версии 9 или выше
- **Доступ к сети:** для установки зависимостей

---

## Шаг 1: Проверка Node.js

Откройте командную строку (PowerShell или CMD) и проверьте версии:

```bash
node -v
npm -v
```

Если Node.js не установлен, скачайте с официального сайта:
👉 https://nodejs.org/ (рекомендуется LTS версия)

---

## Шаг 2: Передача проекта

### Вариант A: Через локальную сеть (рекомендуется)

**На основном компьютере:**
1. Откройте общий доступ к папке проекта
2. Путь к проекту: `C:\Users\direct02\Documents\DIRECT-LINE\Company_site`

**На другом компьютере:**
1. Скопируйте папку проекта в локальную директорию
2. Например: `C:\Projects\Company_site`

### Вариант B: Через архив

1. Запакуйте проект в ZIP-архив
2. Передайте архив по сети
3. Распакуйте на целевом компьютере

---

## Шаг 3: Установка зависимостей

Откройте терминал в папке проекта:

```bash
cd C:\Projects\Company_site
```

Установите все зависимости:

```bash
npm install
```

⏱️ **Время установки:** 2-5 минут (в зависимости от скорости интернета)

---

## Шаг 4: Настройка переменных окружения

Скопируйте файл `.env.local.example` в `.env.local`:

**Windows (PowerShell):**
```bash
copy .env.local.example .env.local
```

**Windows (CMD):**
```bash
copy .env.local.example .env.local
```

### Необходимые переменные

Откройте `.env.local` и проверьте:

```env
# Обязательно
NEXT_PUBLIC_SITE_URL=http://localhost:3000

# Опционально (для форм и CRM)
BITRIX24_WEBHOOK_URL=your_webhook_url
```

---

## Шаг 5: Запуск сайта

### Режим разработки (рекомендуется для тестирования)

```bash
npm run dev
```

✅ Сайт доступен по адресу: **http://localhost:3000**

🔄 **Автоматическая перезагрузка:** При изменении файлов сайт обновляется автоматически

---

### Production режим

```bash
npm run build
npm start
```

✅ Сайт доступен по адресу: **http://localhost:3000**

## Решение проблем

### ❌ Ошибка: "npm is not recognized"

**Решение:** Добавьте Node.js в PATH или переустановите Node.js

### ❌ Ошибка: "Cannot find module"

**Решение:**
```bash
npm install
```

### ❌ Ошибка: "Port 3000 is already in use"

**Решение 1:** Остановить процесс на порту 3000
```bash
# PowerShell
Get-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess | Stop-Process

# Или просто используйте другой порт
npm run dev -- -p 3001
```

### ❌ Ошибка: "ENOSPC: System limit for number of file watchers reached"

**Решение (Linux/Mac):**
```bash
echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
```

## Доступ с других устройств в сети

### На основном компьютере (где запущен сайт):

1. Узнайте локальный IP-адрес:

**Windows (PowerShell):**
```bash
ipconfig
```

Найдите строку **IPv4 Address** (например, `192.168.1.100`)

2. Запустите сайт с доступом по сети:

```bash
npm run dev -- -H 0.0.0.0
```

### На других устройствах в сети:

Откройте браузер и введите:
```
http://192.168.1.100:3000
```

(замените `192.168.1.100` на ваш IP-адрес)

## Брандмауэр Windows

Если сайт не доступен с других устройств:

1. Откройте **Брандмауэр Windows**
2. Выберите **Дополнительные параметры**
3. Создайте **Правило для входящих подключений**
4. Разрешите порт **3000**

Или временно отключите брандмауэр для тестирования.
